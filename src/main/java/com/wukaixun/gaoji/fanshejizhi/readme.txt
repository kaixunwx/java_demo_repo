什么是反射机制？
1.Java反射机制的核心是在程序运行时动态加载类并获取类的详细信息，从而操作类或对象的属性和方法。本质是JVM在获取到class对象后通过反编译获取对象的各种信息
2.Java属于先编译后运行的语言，程序中对象的类型在编译期就确定下来了，而在程序需要在运行时动态加载某些类，这些类因为之前用不到，所以没有被加载到JVM。通过反射，可以在运行时动态得创建对象并调用
其属性，不需要提前在编译期知道运行的对象是谁。

反射机制原理？
Java正常类加载过程是当执行 Student student = new Student()时，会触发JVM加载Student.class文件，JVM从本地磁盘找到Student.class文件并加载到JVM内存中。.class加载完成后，JVM会自动
创建一个class对象，一个类只会产生一个class对象。
反射机制是获取到一个class对象，通过反编译获取到Student对象的各种信息
